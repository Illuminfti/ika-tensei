## 1. Aesthetic Target: "Dark Pixel Occult Anime"

The trailer needs to feel like **Shin Megami Tensei meets a ritual summoning reel**. Reference points:
- SMT / Persona franchise (dark orchestral + chiptune hybrid, Latin/occult choirs)
- Darksynth / Retrowave (industrial distorted synths, pulsing bass)
- Old-school NES/SNES horror RPG (chiptune with minor keys, arpeggios)
- Trailer hype conventions (Hans Zimmer BRAAAM, sub hits, impacts on cuts)

The tone should feel **dangerous, arcane, and exciting** — not cute 8-bit. Dark + powerful first, pixel-art second.

## 5. 30-Second Trailer Audio Structure

### The Arc: Riser → Drop → Build → Climax → Stinger

```
[0:00–0:03]  HOOK / COLD OPEN
  - Silence or single ambient drone
  - Low chime bell / occult tone
  - No music yet, just atmosphere
  - First visual hit: 1 impact SFX

[0:03–0:08]  RISER (tension build)
  - Soft underscore music starts (quiet)
  - Long riser SFX climbing in pitch
  - HP filter sweep (bass cuts out, tension rises)
  - 1–2 whooshes for scene transitions
  - End: FULL HIT on beat 1 of bar 3

[0:08–0:15]  DROP / STATEMENT
  - Music drops IN with force (full arrangement)
  - Bass + drums kick
  - Cuts sync to every beat (4/4 or half-time)
  - Whoosh on each scene cut
  - Braams between major shots

[0:15–0:22]  ESCALATION BUILD
  - BPM edit: faster perceived cuts
  - Sub hits on every 2 beats
  - Choir/orchestral elements swell
  - Second riser leads to final climax
  - Glitch SFX on blockchain/tech reveal

[0:22–0:28]  CLIMAX
  - Loudest, most intense section
  - Fast cuts (every beat or half-beat)
  - Full orchestral + chiptune layered
  - Braams or reverse reverbs between shots
  - Big IMPACT SFX on logo reveal

[0:28–0:30]  STINGER / TAG
  - Music drops out abruptly or fades in 0.5s
  - Single stinger tone (logo sting)
  - Silence = power
  - Optional: low drone continues under logo
```

### Dynamic Contrast Rule

> **Never keep the volume constant.** The power of an impact comes from the **contrast** — drop volume just before a hit so the hit feels 10x bigger. Quiet → explosive = impact. Loud → louder = noise.

Implementation in Remotion:
```tsx
// Volume dip before impact at frame 90 (3s @ 30fps)
<Audio
  src={staticFile('music.mp3')}
  volume={(f) => {
    if (f >= 85 && f < 90) return interpolate(f, [85, 90], [1, 0.2]); // dip
    if (f >= 90 && f < 95) return interpolate(f, [90, 95], [0.2, 1]); // restore
    return 1;
  }}
/>
```

## 6. BPM Matching for Cut Timing

### The Formula

```
Frames per Beat = (FPS × 60) / BPM
Seconds per Beat = 60 / BPM
```

### Common BPM Reference Table (30fps)

| BPM | Frames/Beat | Seconds/Beat | Cuts per 30s | Feel |
|-----|-------------|--------------|-------------|------|
| 80  | 22.5 | 0.75s | ~40 | Slow, heavy, cinematic |
| 100 | 18.0 | 0.60s | ~50 | Mid-pace, dramatic |
| 120 | 15.0 | 0.50s | ~60 | Standard hype |
| 140 | 12.9 | 0.43s | ~70 | Intense, action-forward |
| 160 | 11.25 | 0.375s | ~80 | Very fast, climax |

**Recommended for Ika Tensei:** Start at ~80 BPM feel (slow ominous), push to ~120–140 for climax.

### Remotion Frame Constants for Beat Sync

```typescript
const FPS = 30;
const BPM = 120;

const FRAMES_PER_BEAT = (FPS * 60) / BPM;       // 15 frames
const FRAMES_PER_BAR = FRAMES_PER_BEAT * 4;       // 60 frames (4/4)
const FRAMES_PER_HALF_BEAT = FRAMES_PER_BEAT / 2; // 7.5 frames

// Beat timestamps (frame numbers)
const BEAT_1 = 0;
const BEAT_2 = FRAMES_PER_BEAT;
const BEAT_3 = FRAMES_PER_BEAT * 2;
const BAR_2_START = FRAMES_PER_BAR;

// Helper: snap to nearest beat
const snapToBeat = (frame: number) =>
  Math.round(frame / FRAMES_PER_BEAT) * FRAMES_PER_BEAT;
```

### Cut Timing Strategy

1. **On the beat** — Most reliable for hype trailers
2. **On the downbeat** (beat 1 of each bar) — For heavy/dramatic moments
3. **Off-beat (syncopation)** — Adds energy, use sparingly
4. **Half-time cuts** — During slow build sections (1 cut per 2 beats)
5. **Double-time climax** — Cut every half-beat in final 6s

### Components Available

#### `<Audio>` from `@remotion/media` (Recommended - New)
- Uses Mediabunny for frame-accurate extraction during render
- Better than Html5Audio for precise sync

```tsx
import { Audio } from '@remotion/media';
import { staticFile } from 'remotion';

// Basic music track
<Audio src={staticFile('music/dark-trailer.mp3')} />

// With trim + volume control
<Audio
  src={staticFile('music/dark-trailer.mp3')}
  trimBefore={30}        // skip first 1s (30 frames @ 30fps)
  trimAfter={900}        // stop at 30s mark (900 frames)
  volume={(f) => interpolate(f, [870, 900], [1, 0], { extrapolateLeft: 'clamp' })}
  // Fade out in last 1s
/>
```

### `@remotion/media-utils` — Audio Analysis

```tsx
import { useAudioData, visualizeAudio } from '@remotion/media-utils';
import { useCurrentFrame, useVideoConfig } from 'remotion';

export const AudioReactiveScene = () => {
  const frame = useCurrentFrame();
  const { fps } = useVideoConfig();
  const audioData = useAudioData(staticFile('music/dark-trailer.mp3'));

  if (!audioData) return null;

  // Get frequency spectrum for current frame
  const visualization = visualizeAudio({
    fps,
    frame,
    audioData,
    numberOfSamples: 32, // 32 frequency buckets
  });

  // visualization[0] = bass, visualization[31] = treble
  const bassLevel = visualization[0];  // 0–1
  const midLevel = visualization[8];

  // Use to drive animations
  return (
    <div style={{
      transform: `scale(${1 + bassLevel * 0.2})`, // pulse with bass
      opacity: 0.5 + midLevel * 0.5,
    }}>
      {/* content */}
    </div>
  );
};
```

#### Available `@remotion/media-utils` Functions

| Function | Purpose |
|----------|---------|
| `useAudioData(src)` | Hook: load audio into state, handles delayRender |
| `getAudioData(src)` | Async: fetch audio data (use outside components) |
| `visualizeAudio({fps, frame, audioData, numberOfSamples})` | Get frequency bins for current frame → array 0–1 |
| `visualizeAudioWaveform({fps, frame, audioData, windowInSeconds})` | Get waveform slice around current frame |
| `getWaveformPortion({audioData, startFrom, durationInFrames, fps})` | Get simplified volume data for a range |
| `getAudioDurationInSeconds(src)` | Duration without CORS requirement |

### Volume Automation Patterns

```tsx
// Fade in (first 30 frames)
volume={(f) => interpolate(f, [0, 30], [0, 1], { extrapolateLeft: 'clamp', extrapolateRight: 'clamp' })}

// Fade out (last 30 frames of 900-frame trailer)
volume={(f) => interpolate(f, [870, 900], [1, 0], { extrapolateLeft: 'clamp', extrapolateRight: 'clamp' })}

// Dynamic dip before impact at frame 180
volume={(f) => {
  if (f >= 165 && f < 180) return interpolate(f, [165, 180], [0.8, 0.1]);
  if (f >= 180 && f < 195) return interpolate(f, [180, 195], [0.1, 1.0]);
  return 0.8;
}}

// Sidechain pump effect (ducks on beat)
volume={(f) => {
  const beat = f % FRAMES_PER_BEAT;
  return interpolate(beat, [0, 4, 12], [0.3, 0.8, 0.8], { extrapolateRight: 'clamp' });
}}
```

## 9. Practical Audio Workflow

### Step 1: Pick Track First
Choose music before designing cuts. The BPM determines the entire edit rhythm.

### Step 2: Map Beat Grid
```typescript
// In trailerConfig.ts
export const AUDIO_CONFIG = {
  BPM: 120,
  FPS: 30,
  get framesPerBeat() { return (this.FPS * 60) / this.BPM; },  // 15
  get framesPerBar() { return this.framesPerBeat * 4; },         // 60
  
  // Beat markers (frame numbers)
  beatFrames: Array.from({ length: 32 }, (_, i) => Math.round(i * 15)),
};
```

### Step 3: Assign Cuts to Beat Frames
```typescript
export const CUT_POINTS: CutPoint[] = [
  { frame: 0,   scene: 'cold-open',  sfx: 'chime' },
  { frame: 60,  scene: 'seal-glow',  sfx: 'riser-start' },
  { frame: 90,  scene: 'seal-close', sfx: 'impact' },
  { frame: 105, scene: 'guild-shot', sfx: 'whoosh' },
  { frame: 120, scene: 'chain-map',  sfx: 'whoosh' },
  { frame: 150, scene: 'climax-1',   sfx: 'braaam' },
  { frame: 165, scene: 'climax-2',   sfx: 'whoosh-heavy' },
  { frame: 180, scene: 'cto',        sfx: 'impact-big' },
  { frame: 810, scene: 'logo',       sfx: 'stinger' },
];
```

### Step 4: Layer in Remotion
```tsx
// TrailerAudio.tsx
export const TrailerAudio: React.FC = () => (
  <>
    <Audio src={staticFile('music/main-track.mp3')} volume={0.75} />
```
